# Evaluation Rubric

Your capstone project will be evaluated across four main categories totaling 100 points.

| Category | Points | Weight |
|----------|--------|--------|
| Technical Implementation | 40 | 40% |
| Physical AI Concepts | 30 | 30% |

| **Total** | **100** | **100%** |

## ‚ú® Bonus Points (up to +5)

- **+3**: Deployment on real hardware (if project was in simulation)
- **+2**: Exceptional documentation (tutorial-quality)
- **+2**: Open-source contribution (published to GitHub with proper docs)
- **+2**: Comparison with alternative approaches
- **+1**: Video tutorial explaining your work

*Maximum bonus: 5 points (cannot exceed 100 total)*

## üìã Self-Assessment Checklist

Before submission, check:

### Technical (40 pts)
- [ ] Code is clean and follows style guidelines
- [ ] Architecture is modular and clear
- [ ] All features work as intended
- [ ] System handles errors gracefully

### Physical AI (30 pts)
- [ ] Sense-think-act loop is evident
- [ ] Multiple sensors integrated
- [ ] Actuators controlled properly
- [ ] Control strategy is appropriate

### Documentation (20 pts)
- [ ] Technical report is complete
- [ ] Code has comments and docstrings
- [ ] README explains setup and usage
- [ ] Demo video is clear

### Innovation (10 pts)
- [ ] Project is appropriately challenging
- [ ] Some creative solutions used

---

## üí° Tips for High Scores

**Code Quality**:
- Use linters (pylint, flake8)
- Follow consistent style
- Write small, focused functions
- Use meaningful names

**Documentation**:
- Start early, document as you go
- Include diagrams
- Show results with plots
- Explain design decisions

**Demonstration**:
- Record multiple runs
- Show success and failure cases
- Explain what's happening
- Highlight key features

**Innovation**:
- Try something new or different
- Explain why your approach is good
- Compare with alternatives
- Go beyond minimum requirements
